- FINDING: Per-mode git argv shapes duplicated between the preflight probes (run.go) and the L1 diff sources (generate.go)
  SEVERITY: high
  FILES: internal/commit/run.go:739-749 (stagedProbeArgs/trackedProbeArgs/untrackedProbeArgs), internal/commit/generate.go:168,185,239 (stagedDiff/trackedWorktreeDiff/untrackedFiles)
  DESCRIPTION: Two files independently encode the SAME three per-mode git argv "shapes" that must stay byte-aligned. The preflight builders emit `["diff","--cached","--name-only","--","."]+excludes`, `["diff","HEAD","--name-only","--","."]+excludes`, `["ls-files","--others","--exclude-standard","--","."]+excludes`; the generate sources emit the identical argv minus `--name-only` (`["diff","--cached","--","."]+excludes`, `["diff","HEAD","--","."]+excludes`, `["ls-files","--others","--exclude-standard","--","."]+excludes`). The `-- .` separator, the `:(exclude)` excludes tail, and the per-mode base verb/refspec are duplicated across the two files. The authors flagged the hazard themselves in-code: run.go:734-738 and 700-712 state the preflight argv must be "identical to generate's argv" so "the preflight and the AI's L1 diff read one exclusion-filtered source." That invariant is enforced only by two copies agreeing — exactly the copy-paste-drift risk that emerges when probe and source were written as separate concerns. A future edit to one base (e.g. swapping `git diff HEAD` for `git diff --merge-base HEAD`, or reordering the exclude tail) silently breaks the "one source" guarantee and the preflight emptiness verdict diverges from what the AI actually sees.
  RECOMMENDATION: Extract the three per-mode base argv into one shared set of builders (e.g. unexported `stagedBaseArgs/trackedBaseArgs/untrackedBaseArgs` returning the `[verb, refspec/flags, "--", "."]` prefix, with `excludePathspecs` appended by callers). The preflight probe builders append `--name-only` to the diff cases; the generate sources use the prefix as-is. This puts the per-mode shape (and the `-- .` + excludes contract the in-code comments demand stay identical) in exactly one place, so the "preflight reads the same exclusion-filtered source as the L1 diff" invariant is structural rather than comment-enforced.

- FINDING: StagingMode dispatch switch duplicated between wouldStageNothing (run.go) and sourceDiff (generate.go)
  SEVERITY: medium
  FILES: internal/commit/run.go:715-732 (wouldStageNothing), internal/commit/generate.go:144-153 (sourceDiff)
  DESCRIPTION: Both functions switch on StagingMode with the SAME three-way structure and the SAME AddAll special case (tracked first, short-circuit, then untracked). sourceDiff routes All→trackedWorktreeDiff, AddAll→addAllWorktreeDiff (tracked + untracked), default→stagedDiff; wouldStageNothing routes All→tracked probe, AddAll→tracked-then-untracked probe with an early-out, default→staged probe. run.go:698-712 explicitly notes each probe branch "mirrors" its generate counterpart.   RECOMMENDATION: This is the same drift surface as the argv finding and is best consolidated alongside it. Centralise the per-mode definition (which sources/probes compose for each mode, including the AddAll tracked+untracked composition) so both the diff path and the emptiness path derive their mode behaviour from one description rather than two parallel switches. If full unification is too invasive, at minimum co-locate the two switches' mode mapping behind a shared mode-descriptor so the AddAll short-circuit rule lives once.

Low impact (small blocks) but it removes the third+ copies of the wrap-string-with-command-name pattern and keeps error wording consistent.
