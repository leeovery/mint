- FINDING: Four near-identical body/regenerator producer closures duplicate the source-dispatch across single and batch cmd paths
  SEVERITY: low
  FILES: cmd/mint/regenerate_run.go:59-82, cmd/mint/regenerate_all.go:95-117
  DESCRIPTION: newRegenerateBodyProducer/newBatchBodyProducer and newRegenerateRegeneratorProducer/newBatchRegeneratorProducer are four closures that all encode the identical reuse-vs-fresh dispatch rule (reuse → ReadReuseBody/nil regenerator; fresh → RegenerateFreshBody/RegenerateFreshRegenerator). The single and batch variants differ only in whether the version.Resolution is closed over up front (single) or passed per call (batch). The dispatch decision is the load-bearing rule, and it is authored four times. If a third source were ever added, or the reuse/fresh body contract changed, all four sites must be updated in lockstep. The batch forms are the more general shape (Resolution as a parameter); the single forms could be expressed by binding that parameter, collapsing four functions to two.
  RECOMMENDATION: Define the two batch-shaped producers (keyed on version.Resolution) as the canonical pair, and derive the single-version closures by partially applying the fixed Resolution, so the reuse-vs-fresh dispatch lives in exactly one place per concern.
- FINDING: ResolvePublisher surfaces resolution failures under the literal "preflight" stage while regenerate's own preflight failures use a named constant
  SEVERITY: low
  FILES: internal/engine/release.go:935, internal/engine/regenerate.go:27
  DESCRIPTION: engine.ResolvePublisher (the shared regenerate-path publisher resolver) surfaces a non-downgrade resolution error via surface(deps.Presenter, regeneratePreflightStage, err) — correct — but regeneratePreflightStage is a const defined in regenerate.go as "preflight", and the forward-path publisher resolution in Release() surfaces the same class of failure with a bare string literal "preflight". The two paths agree on the stage label only by coincidence of the literal matching the constant's value. Minor seam-consistency wrinkle: the forward and regenerate publisher-resolution paths are deliberately kept behaviourally identical, yet one references the shared constant and the other a string literal, so a future relabel of the regenerate preflight stage would silently diverge the two.
  RECOMMENDATION: Have the forward-path publisher-resolution surface sites reference the same stage constant (or document that the forward path intentionally owns its own "preflight" literal), so the stage label cannot drift between the two mirrored resolution paths.
